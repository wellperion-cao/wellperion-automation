2. 노션 폴링 watcher 5종 → 가이드허브 시드/큐 전환
================================================================
위치: telegram_bot\*.py (daily_scheduler 가 잡으로 등록).
공통 알림 대체 원칙: 노션 상태 변경 감지 → 가이드허브 시드(status·startDate)
  + status\_queue.json·status\{role}.json 변화 감지로 치환.

(1) status_change_watcher  (1분 주기)
    - 현재: 업무자동화DB '상태' 변경 → 텔레그램 C-Level 상태 알림.
    - 전환: status\_queue.json / status\{role}.json 의 항목 status 필드
      변화 감지(파일 mtime + 직전 스냅샷 비교) → 동일 텔레그램 알림.
    - GM 알림 대체: "C-Level 상태 변경" 알림을 큐 JSON diff 로 생성.
    - 단계: ①큐 스키마 확정 ②스냅샷 비교 함수 ③기존 포맷 재사용 ④병행가동 1주.

(2) archive_result_watcher  (5분, 1단계에서 이미 비활성)
    - 전환: 결과물DB '결과 보고'·'유지보수'·'완료' → 가이드허브 시드 status
      '완료' + 결과물 포인터로 치환. PSP 자동 생성(노션 페이지 생성)은
      가이드허브에 SSOT 정립 섹션이 없으므로 보류/재설계.
    - GM 알림 대체: 시드 status '완료' 전환 시 1줄 완료 알림(이미 21시 보고가
      가이드허브 시드 기반 → 중복 최소화 위해 일일 통합만 유지 검토).

(3) planning_to_archive_watcher  (5분)
    - 현재: Start 기획DB → 결과물DB 이관 감지.
    - 전환: 가이드허브 2단계 파이프라인(기획→운영 아카이브)이 이미 시드 기반이면
      watcher 자체 폐기. 노션 이관 이벤트가 더 이상 없으므로 제거 후보 1순위.

(4) auto_task_watcher  (5분)
    - 현재: 업무자동화DB 상태='진행중' + 실행시간 ±10분 → 자동 실행 트리거.
    - 전환: 가이드허브 시드 status='진행중' + startDate/실행시각 윈도우로 치환.
      트리거 소스를 노션 쿼리 → 가이드허브 시드 파서(_fetch_*_from_guidehub
      패턴 재사용)로 교체.
    - GM 알림 대체: pre_task_notifier(H-15분 사전 알림)도 동일 소스로 교체.

(5) permission_watcher  (15분)
    - 현재: 노션 통합 권한 상시 감시.
    - 전환: 노션 의존 자체가 권한 점검 대상 → 노션 사용 중단 시 이 watcher 폐기.
      대체 불필요(노션 미사용이면 권한 감시 무의미). 제거 후보.

전환 공통 단계(각 watcher):
  A. 가이드허브/큐 소스로 동일 출력 만드는 함수 작성(노션 함수와 1:1).
  B. daily_scheduler 잡을 새 함수로 교체(코드 레벨, if 토글로 병행).
  C. 1주 병행 가동 → 알림 동등성 확인 → 노션 경로 코드 제거.
  D. 각 단계 daily_scheduler 재시작은 GM 결정(프로세스 kill 금지).
