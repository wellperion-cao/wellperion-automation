AI C-Level 협업 매뉴얼

웰페리온의 AI C-Level(CEO · COO · CFO · CMO · CTO · CHRO · CPO)은 매일·매주 자동으로 돌아가며 Notion에 리포트와 데이터베이스를 갱신합니다. 이 매뉴얼은 매니저가 각 AI C-Level의 산출물을 어디서 보고, 어떻게 질의하고, 어떻게 검수하고, 언제 사람 의사결정자에게 가져갈지를 정리한 실사용 가이드입니다.

Document v1.0 Audience Manager Channel Notion Updated 2026.04
The Principle

AI C-Level은 매니저를 대체하는 존재가 아니라, 매니저의 시야를 넓혀주는 초안 작성자입니다.

AI가 데이터를 모으고 패턴을 제시하면, 매니저가 맥락을 더해 결정을 완성합니다. 이 매뉴얼은 그 협업의 리듬을 정리합니다.

Common · Do Not Input

AI에 절대 입력하지 않는 정보

AI 세션과 Notion DB에는 다음 정보를 직접 입력하지 않습니다. 꼭 필요한 경우 해시·마스킹·코드로 대체합니다.

  • 회원 개인정보 주민등록번호, 실제 생년월일, 주소, 여권번호. 대신 회원 ID 또는 이름+연락처 뒷자리.
  • 금융정보 카드번호, 계좌번호, CVC. 결제 관련은 벤더 관리 코드만 기록.
  • 건강 상세정보 진단명·처방·수술이력. 필요시 "건강 고지 있음 (레벨 1–3)" 수준으로 요약.
  • 직원 개인정보 급여 상세, 고용계약 조항, 징계 이력. HR AI에게도 코드화된 레퍼런스만.
  • 법적 분쟁 진행중 내용 사실관계·상대방 식별정보 금지. 반드시 오프라인으로 처리.
Violation Rule 위 항목이 이미 AI 세션에 들어간 것을 발견하면 즉시 해당 대화를 종료하고 CTO/GM에 공유. 세션 히스토리는 보관 주기 내 파기 요청.

전사 주간 총괄 보고

  • Top 3 로우라이트 가장 아쉬운 일 3가지 · 원인 가설 + 다음 주 조치 제안
  • 주간 리스크 맵 발생 가능성 × 영향도 2×2 · 상위 5개 리스크 항목

02결과 검수 체크리스트

AI 리포트를 그대로 본부장·대표에게 올리기 전에 매니저가 확인할 6가지.

  • 숫자 정합성 인용된 지표가 원본 C-Level 리포트와 일치하는가. 수치 반올림 오차 확인.
  • 맥락 누락 AI가 놓친 이번 주 특수 상황(휴일·이벤트·사고 등)이 있는지 직접 추가.
  • 중복 제거 두 C-Level 리포트에 같은 이슈가 중복되어 올라왔는지 확인.
  • 톤 조정 대표 보고용이므로 추정·단정 표현 → 가능성 표현으로 조정.
  • 실행 주체 제안된 조치에 실제 담당자가 배정되어 있는지 확인. 배정 없으면 반려.
  • 민감 정보 개인정보·재무 상세·법적 분쟁 관련 문구가 남아있는지 최종 스캔.

Q-CEO-03리스크 재평가 필요할 때
현재 리스크 맵에서 "[리스크 항목]"의 영향도를 재평가해줘. 근거: [최근 발생한 이벤트/데이터]. 재평가 후 변경된 2×2 맵을 보여줘.

Q-CEO-05대표 보고 직전 최종 정리

When to Escalate

AI가 답할 수 있는 영역과 사람에게 가져가야 할 영역을 구분합니다.

AI 주간 지표 요약, 과거 데이터 비교, 리포트 재구성, 실행 리스트 생성 Self-serve
Human 전사 우선순위 변경, 조직 구조 관련 의사결정, 외부 파트너 교체 GM · 대표
Stop 법적 이슈, 언론 노출, 안전사고, 이사회 사안 — AI 판단에 의존 금지 대표 직보
Troubleshooting ① 리포트가 금요일 17:00까지 생성되지 않은 경우 — 각 C-Level 세션이 정상 종료됐는지 확인 후 수동 재실행. ② 숫자가 다른 C-Level 리포트와 어긋나는 경우 — 원본 리포트의 Snapshot 시각 확인 (AI가 이전 스냅샷을 참조했을 가능성). ③ 같은 질문에 다른 답이 나오는 경우 — 컨텍스트 차이 가능성. 동일 질문을 하나의 세션에서 연속으로 재시도.
Chapter Ⅱ · COO

일일 운영 모니터링

매일 오픈 전 시설·예약·컴플레인 상태를 요약해 Notion에 올려주는 AI. 매니저의 플로어 워크 직전 15분을 대시보드 점검으로 쓰면, 공간이 보내는 신호를 놓치지 않습니다.

Role
시설·예약·컴플레인 통합 대시보드전일 이상 로그 · 오늘 예약률 · 컴플레인 Top 3

01산출물 구성

  • 시설 이상 로그 온도·습도·수질·음향 기준 이탈 기록 + 발생 시각 + 조치 여부
  • 오늘 예약 현황 시간대별 예약률 · 피크 예상 시간 · 대기 회원 리스트
  • 컴플레인 Top 3 전일 접수된 컴플레인 중 중요도 순 3건 + 반복 여부 표시
  • 근무표 · 인원 상태 오늘 근무자 · 결근·지각 · 스페셜 케어 필요 회원의 담당 트레이너
  • 오픈 체크리스트 완료율 전일 마감 · 오늘 오픈 준비 작업 진행 상태

02결과 검수 체크리스트

  • 현장 대조 대시보드 숫자와 플로어 워크에서 실제 관찰한 상태가 일치하는지.
  • 컴플레인 반복 AI가 "반복"으로 표시하지 않았지만 내 기억에 있는 반복 이슈가 있는지.
  • 예약 이상치 평소와 다른 예약 패턴(급증·급감)이 있으면 원인 질의.
  • 인원 공백 피크 시간대에 인원이 부족한 섹션이 없는지 확인.
  • 누락 장비 장비 가동률 100% 표시여도 실제 고장 신호가 있었는지 트레이너에게 더블체크.

일일 비용 모니터링

전일 지출을 카테고리별로 분해하고, 월 예산 대비 소진률과 이상 지출을 플래그로 표시하는 AI. 매니저는 플래그된 항목의 해명만 달아도 월말 리포팅 시간이 크게 줄어듭니다.

  • 이상치 플래그 전월 평균 ±30% 이탈 항목에 자동 ⚠️ 표시

  • 플래그 해명 ⚠️ 표시된 항목에 24시간 내 1줄 해명 메모 추가.

  • 중복 결제 같은 벤더·같은 금액이 7일 내 반복됐는지 수동 확인.
  • 증빙 첨부 전일 지출의 세금계산서·현금영수증 연결 상태 점검.
  • 환율 변동 해외 벤더 결제가 있다면 환율 기준일 확인 (AI가 당일 환율로 계산했는지).

매일 오전 SNS·채널별 콘텐츠 아이디어를 3–5개 초안으로 생성하는 AI. 매니저는 초안 카드를 빠르게 검수해 승인·반려·수정 중 하나로 판정하고, 반영된 피드백이 내일 AI 품질로 돌아옵니다.

Role
SNS·뉴스레터·내부 공지 초안 생성채널별 톤 조정 · 이미지 방향 · 업로드 시간 제안
Schedule
매일 09:00 자동 실행주 1회 월요일 주간 테마 리셋
Notion Location
Content CalendarDB · Status: Draft / Approved / Published

01산출물 구성

  • 콘텐츠 카드 3–5개 채널 · 테마 · 캡션 초안 · 비주얼 방향 · 권장 업로드 시간
  • 주간 테마 그 주를 관통하는 키 메시지 1줄 + 서브 테마 3개
  • 회원 피드백 요약 전일 댓글·DM·후기 중 브랜드에 인용 가능한 발언 (동의 필요 표시)
  • 경쟁 브랜드 노트 벤치마크 3곳의 최근 24시간 주요 포스팅 요약
  • 브랜드 톤 체크 초안 카드가 브랜드 가이드 톤 앤 보이스 3기준을 통과했는지 자체 점수

02결과 검수 체크리스트

  • 수식어 점검 "프리미엄·최고·완벽" 같은 포괄 수식이 남아있지 않은지.
  • 회원 개인 노출 AI가 제안한 회원 후기 인용에 서면 동의가 있는지 확인.
  • 사실 검증 수치·일정·장소가 실제 운영 데이터와 일치하는지.
  • 이미지 방향 일관성 자연광·보정 ±5%·공간 우선 3기준 위반 여부.
  • 채널별 톤 인스타그램 · 뉴스레터 · 내부 공지의 톤 구분이 살아있는지.

주간 시스템 점검

회원관리(브로제이) · 예약 · 결제 · 출입 시스템의 상태를 주간 단위로 진단하고, 장애 이력·정합성·백업 상태를 한 페이지로 올려주는 AI. 매니저는 Severity 등급만 보면 무엇을 먼저 처리할지 결정할 수 있습니다.

Role
시스템 헬스 리포트 · 데이터 정합성 감사Severity 1–4 분류 · 백업 복원 훈련 결과

01산출물 구성

  • 주간 장애 이력 Severity · 발생 시각 · 복구 시각 · 원인 · 재발 방지 조치
  • 데이터 정합성 회원 중복 레코드 · 보호자 미연결 유소년 · 결제 실패 미처리 건수
  • 성능 지표 주요 화면 로딩 시간 · 예약 성공률 · 결제 성공률 주간 평균
  • 보안 알람 비정상 접근 · 권한 초과 · 대량 다운로드 시도 로그
  • 백업 상태 일일 백업 성공률 + 분기 복원 훈련 결과

02결과 검수 체크리스트

  • Severity 분류 확인 AI가 Sev 1로 분류한 건이 실제 회원 영향이 있었는지 더블체크.
  • 원인 가설의 깊이 "원인 미상"으로 끝난 건은 반드시 후속 조사 티켓 생성.
  • 백업 복원 실제 시험 AI의 "백업 성공" 지표가 복원까지 검증된 것인지 분기 1회 직접 실행.
  • 외주 벤더 영향 브로제이 장애인지 자체 인프라 장애인지 구분 확인.
  • 보안 알람 후속 비정상 접근 로그가 있으면 해당 계정·IP 차단 여부 확인.

일일 HR 모니터링

출결·근태·1:1 일정·기념일을 매일 아침 Notion에 정리해주는 AI. 매니저는 팀원의 컨디션 신호를 조기에 잡고, 1:1 전에 맥락을 한 번에 훑을 수 있습니다.

Role
출결·근태·1:1·기념일 통합 브리프개인 데이터는 코드화된 형태만 저장
Schedule

Notion Location
HR Pulse본인 팀원 필터 뷰로 접근

01산출물 구성

  • 오늘 근태 요약 출근 · 지각 · 결근 · 휴가 · 외근
  • 연차 잔여 분기 소진률 하위 3인 + 소진률 상위 3인
  • 이번 주 1:1 스케줄 예정 대화 + 지난 1:1 메모 요약
  • 기념일·생일 이번 주 축하 대상자 + 팀 내 축하 방식 제안
  • 컨디션 신호 최근 3주간 근태·피드백 패턴에서 AI가 읽은 변화 (주의 → 확인 권유)

02결과 검수 체크리스트

  • 개인정보 최소화 AI 답변에 주민번호·주소·계좌가 포함되면 즉시 세션 종료.
  • 추정 주의 "컨디션 신호"는 사실이 아니라 추정치. 본인과 대화로만 확인.
  • 1:1 메모 일관성 지난 메모의 약속 사항이 이번 주에 이어지고 있는지.
  • 편향 확인 AI가 특정 팀원에게 반복적으로 부정 신호를 주는지 — 팀원 간 형평성 체크.
  • 근태 데이터 원본 출근 시스템 원본과 AI 요약이 일치하는지 주 1회 샘플링.
